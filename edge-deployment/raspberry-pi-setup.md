# Edge Deployment Environment

This document describes the edge deployment environment used in the experimental evaluation of the CAMS+GenAI framework.

## Edge Infrastructure

The experimental environment includes Raspberry Pi devices acting as edge gateways. These nodes are responsible for processing contextual data streams and executing adaptation logic close to the data sources.

Edge nodes provide several advantages for context-aware systems:

- reduced latency
- local data processing
- improved resilience under intermittent connectivity

## Hardware Configuration

The deployment environment includes the following components:

- Raspberry Pi 4 devices used as edge gateways
- Android mobile devices running generated applications
- IoT sensors providing contextual data
- Wi-Fi network connecting mobile and edge nodes

## Communication Infrastructure

Communication between system components is implemented using the MQTT protocol.

MQTT provides a lightweight publish–subscribe messaging mechanism that enables efficient communication between:

- mobile devices
- IoT sensors
- edge gateways

## Edge Processing

Edge nodes perform several runtime tasks:

- context aggregation
- rule evaluation
- adaptation triggering
- telemetry collection

This architecture enables context-aware applications generated by the CAMS+GenAI framework to operate efficiently in distributed IoT environments.
