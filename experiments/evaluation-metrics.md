# Experimental Evaluation Metrics

This document describes the evaluation metrics used to assess the performance of the CAMS+GenAI framework.

The evaluation focuses on measuring the effectiveness of prompt-driven model synthesis and adaptive deployment in distributed edge–cloud environments.

## Model Accuracy

Model accuracy measures the correctness of DSL models generated by the generative modeling pipeline. Generated models are compared with reference models defined by domain experts.

Accuracy is computed as the proportion of correctly generated modeling constructs relative to the expected DSL structure.

## Prompt Consistency Score (PCS)

The Prompt Consistency Score evaluates the syntactic and semantic validity of LLM-generated DSL artifacts.

PCS is defined as:

PCS = α * Ssyntax + β * Ssemantic + γ * Sconstraint

Where:

- Ssyntax evaluates compliance with the DSL grammar
- Ssemantic evaluates semantic consistency of generated rules
- Sconstraint evaluates adherence to platform constraints

## Response Time

Response time measures the time required to generate DSL models from structured prompts and produce deployable application artifacts.

## Resource Usage

Resource usage measures the computational resources required during model synthesis and application execution in edge environments.

## Adaptation Rate

Adaptation rate measures the frequency with which the system successfully adapts to contextual changes detected by sensors and IoT devices.

## Baseline Frameworks

The experimental evaluation compares the proposed CAMS+GenAI framework with previous frameworks from the CAMS family:

- CAMS-F
- CAMS-Edge

These baselines enable assessing the benefits of integrating generative AI into model-driven engineering pipelines.
